Sequence unpacking

Sequence unpacking#

Python has a very nice trick for separating out items from a sequence, called unpacking.

You unpack a sequence when you separate elements out into separate variables. It is easiest to explain by example.

Consider this tuple, on the right hand side of an assignment:

# a gets the value (1, 2)
a = (1, 2)
a

(1, 2)

We do unpacking when we write a tuple on the left hand side of an assignment.

In the cell below, we are unpacking the two values in a by writing a tuple of variables on the left hand side:

(b, c) = a

This sets the value of b to be the first element in a:

b

1

The value of c is the second element in a:

c

2

You may by now have noticed that, in Python, we can often write tuples without the parentheses:

# In this context, Python assumes you mean the right hand side to be tuple.
my_tuple = 10, 12
my_tuple

(10, 12)

In fact, you can do the same trick when writing your tuple on the left hand side, for unpacking. You can omit the parentheses.

# Omit parentheses in tuple unpacking.
first_val, second_val = my_tuple

The expression on the right hand side can be anything that Python can get a sequence of values from. For example, it can be a tuple, as we have seen, or a list, or an array, or any abstract iterable thing.

# List
d, e = [100, 112]
print('d is', d)
print('e is', e)

d is 100
e is 112

# Array
import numpy as np

f, g = np.array([100, 112])
print('f is', f)
print('g is', g)

f is 100
g is 112

Your tuple on the left hand side can be any length. For example, here we unpack four values from an array:

h, i, j, k = np.arange(4)
print('h is', h)
print('i is', i)
print('j is', j)
print('k is', k)

h is 0
i is 1
j is 2
k is 3

If the length of your left hand side tuple does not match the right hand side, Python will error:

# Four elements on the right, three on the left.
m, n, p = np.arange(4)

---------------------------------------------------------------------------
ValueError                                Traceback (most recent call last)
Cell In [10], line 2
      1 # Four elements on the right, three on the left.
----> 2 m, n, p = np.arange(4)

ValueError: too many values to unpack (expected 3)

Tuple unpacking is particularly useful for returning more than one thing from a function.

Notice that this function returns a tuple with two elements.

def first_func():
    return 7, 8

# The result is a tuple.
result = first_func()
result

(7, 8)

We can immediately separate these two things into two variables, using tuple unpacking:

q, r = first_func()
print('q is', q)
print('r is', r)

q is 7
r is 8

Of course, from the logic you have already seen, the cell above is the same as:

results = first_func()
q = results[0]  # Get the first element of the returned tuple.
r = results[1]  # Get the second element of the returned tuple.
print('q is', q)
print('r is', r)

q is 7
r is 8

There is also a nice syntax to separate the first few elements into variables, but leave the remainder in a tuple.

For example, imagine we are returning 10 things:

def ten_func():
    return np.arange(10)

We could always separate these 10 things into 10 variables, like this:

a, b, c, d, e, f, g, h, i, j = ten_func()
print('a is', a)
print('j is', j)

a is 0
j is 9

Perhaps we only need the first two elements as their own variables, we are happy to pass the rest into a tuple for later use. We can do that with:

k, m, *rest = ten_func()
print('k is', k)
print('m is', m)
print('rest is', rest)

k is 0
m is 1
rest is [2, 3, 4, 5, 6, 7, 8, 9]
